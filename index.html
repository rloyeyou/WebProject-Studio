📖 WebProject Studio 사용 가이드
1
WebProject Studio란?
웹 프로젝트 기획·설계 전 과정을 하나의 HTML 파일에서 처리할 수 있는 올인원 도구입니다. 설치 없이 브라우저에서 바로 실행되며, 모든 데이터는 브라우저(localStorage)에 저장되어 새로고침해도 유지됩니다.
파일 하나로 전부 해결. 별도 앱·서버·인터넷 연결 없이 작동합니다. 바탕화면이나 작업 폴더에 두고 브라우저로 여세요.
2
상단 탭 4개 — 각 탭의 역할
📊 대시보드 — sitemap 표를 붙여넣으면 전체 진행률을 자동 계산·시각화
✨ AI 프롬프트 — 프로젝트 설정 4단계 입력 → Claude·Figma AI·MD 파일 동시 생성
📄 페이지 생성 — 개별 페이지 MD 파일을 폼 입력만으로 즉시 생성
🗂 MD Vault — 브라우저 내 마크다운 문서 관리 (작성·편집·미리보기)
3
좌측 사이드바 — 프로젝트 히스토리
작업한 내용은 좌측 사이드바에 프로젝트 히스토리로 자동 저장됩니다. 최대 30개 프로젝트를 저장할 수 있으며, 언제든 클릭해 불러올 수 있습니다.
히스토리 사용법:
• 새 프로젝트 버튼 → 새 항목 생성 + 이름 편집 모드 자동 진입
• 히스토리 항목 클릭 → 해당 프로젝트 불러오기
• 히스토리 항목 더블클릭 → 프로젝트 이름 직접 수정
• 항목 우측 × 버튼 → 해당 프로젝트 삭제
• 이름 옆 ● 점 → 자동저장 상태 표시 (🟡 저장 중 → 🟢 저장 완료)
4
실시간 자동저장 — 입력하는 족족 저장됩니다
AI 프롬프트 탭에서 값을 변경하면 0.8초 후 현재 활성 프로젝트에 자동 저장됩니다. 페이지·섹션 추가/삭제, 드래그 순서 변경 시에도 즉시 저장됩니다.
단, 활성 프로젝트가 없을 때는 자동저장이 작동하지 않습니다. 새 프로젝트를 먼저 만들거나 히스토리에서 불러온 뒤 작업하세요.
5
슬롯 시스템 — 폴더별로 정리해서 보관
AI 프롬프트 탭 상단의 📂 슬롯 관리를 열면 폴더를 만들고 그 안에 설정값 전체를 슬롯으로 저장할 수 있습니다. 히스토리와 별도로, 완성된 세팅을 영구 보관할 때 사용합니다.
히스토리 vs 슬롯 차이:
• 히스토리 — 작업 이력 자동 기록, 최대 30개
• 슬롯 — 완성된 설정을 폴더별로 직접 정리·보관, 최대 100개/폴더
1
먼저 T51 샘플로 테스트해보기
"📌 T51 샘플 불러오기" 버튼을 누르면 실제 프로젝트 데이터가 자동으로 채워집니다. 전체 진행률 게이지, 완료·진행·미착수 카운트, 페이지별 상태 테이블이 한눈에 표시됩니다.
2
실제 sitemap 데이터 붙여넣기
관리 중인 sitemap 파일에서 페이지 상태표 전체를 복사해서 텍스트 박스에 붙여넣고 "🔍 파싱해서 대시보드 생성"을 클릭합니다.
붙여넣어야 할 형식
| ID | 페이지명 | 섹션 수 | 상태 | 우선순위 | 비고 |
|----|---------|--------|------|--------|------|
| 01 | Home | 9 | ✅ done | P1 | |
표 전체를 그대로 복사·붙여넣기 하면 됩니다.
3
상태값 종류
사용 가능한 상태값
✅ done 또는 done → ✅ 완료
🚧 wip 또는 wip → 🚧 진행중
⬜ todo 또는 todo → ⬜ 미착수
🔴 hold 또는 hold → 🔴 보류
4
페이지 필터 & 보고서 다운로드
대시보드 생성 후 전체 / ✅ 완료 / 🚧 진행중 / ⬜ 미착수 버튼으로 페이지를 필터링할 수 있습니다. "📥 진행 보고서 다운로드"를 누르면 progress-report-날짜.md 파일로 저장됩니다. 클라이언트 공유 또는 팀 보고에 활용하세요.
1
먼저 "📌 T51 Home 프리셋"으로 구조 파악하기
스텝 4(푸터 설정) 하단의 "📌 T51 프리셋" 버튼을 클릭하면 T51 Home 페이지·9개 섹션·T51형 푸터가 자동 세팅됩니다. "✨ 프롬프트 생성"을 눌러보면 어떤 결과가 나오는지 바로 확인할 수 있습니다.
2
4단계 스텝 탭으로 순서대로 입력
좌측 설정 패널은 ① 기본 설정 → ② 페이지 구성 → ③ 섹션 구성 → ④ 푸터 설정 4단계로 나뉩니다. 상단 탭을 클릭하거나 하단 "다음 →" 버튼으로 이동합니다. 완료된 단계는 탭 번호가 초록 ✓로 바뀝니다.
탭은 언제든 자유롭게 이동 가능합니다. 순서를 지키지 않아도 되고, 중간에 수정해도 됩니다. 입력한 모든 값은 프롬프트 생성 시 자동 반영됩니다.
3
① 기본 설정 — 프로젝트 전체 규칙 정의
프로젝트명 / 클라이언트명 / 폰트 / 기준 해상도 / 기준 비율
와이어프레임 스타일 / 반응형 기준 / 색상 테마 / GNB 유형
인터랙션 수준 / 이미지 처리 / 텍스트 처리 / 접근성 레벨
클라이언트 특이사항 (자유 입력)
GNB 유형 선택 팁: T51형 프로젝트는 "햄버거 + 슬라이드 패널 우측"을 선택하세요. ← 표시로 구분됩니다.
4
② 페이지 구성 — 여러 페이지를 한 번에 등록
페이지 ID + 페이지명 + 특이사항을 입력하고 "+ 추가" 버튼(또는 Enter)을 눌러 페이지를 추가합니다. 여러 페이지를 추가한 뒤 프롬프트를 생성하면 모든 페이지가 한 번에 포함됩니다.
섹션 구성(③)은 추가된 모든 페이지에 공통 적용됩니다. 페이지마다 섹션이 다르다면 페이지를 1개씩 추가해서 따로 생성하세요.
5
③ 섹션 구성 — 추가 후 드래그로 순서 변경
섹션 유형 드롭다운에서 유형 선택 → 특이사항 입력 → "+ 추가"(또는 Enter). 추가된 섹션은 왼쪽 ⠿ 핸들을 드래그해서 순서를 바꿀 수 있습니다. × 버튼으로 삭제합니다.
특이사항 작성 팁: 슬래시(/)로 구분하면 읽기 편합니다.
예: Hero 비주얼 / 100%×600px / 오버레이 좌하단
6
④ 푸터 설정 — 글로벌 푸터 스펙 지정
푸터 유형·높이·로고·SNS 아이콘 수·개인정보처리방침 포함 여부를 선택합니다. "커스텀"을 선택하면 직접 스펙을 입력할 수 있습니다. 설정값이 프롬프트에 글로벌 푸터 스펙으로 포함되어 모든 페이지에 공통 적용됩니다.

Figma AI용

1
프리셋으로 Works·Story 페이지 즉시 생성

2
새 페이지 직접 만드는 법
왼쪽 폼에 페이지 ID·페이지명·섹션 수·상태·우선순위·특이사항·레이아웃 유형을 입력하고, 섹션 구성을 추가한 뒤 "📄 파일 생성"을 누릅니다.
페이지 ID: 02 / 페이지명: About Us / 섹션 수: 4
상태: 🚧 wip / 우선순위: P1 / 특이사항: Contact Us 섹션 포함하지 않음
→ 📄 파일 생성 → 오른쪽에 MD 미리보기 → 📥 다운로드 또는 📋 복사
